# MapGIS 矢量文件格式解析说明（*.wt / *.wl / *.wp）

MapGIS 矢量文件是中地数码公司开发的一套二进制矢量数据格式，广泛应用于国内地质调查、工程勘察及科研领域。该格式为闭源格式，文件本身**不携带完整的坐标系描述**，投影与椭球参数需依赖 MapGIS 安装环境（如 `mapgis67\program\ellip.dat`）中的索引表来解释。

本文基于对 MapGIS 6x/67 格式 `.wt`（点）、`.wl`（线）、`.wp`（区）文件的逆向解析实践，以及技术文章《MapGIS 文件解析》中给出的字节级结构描述，整理出可供开源读取器实现参考的格式说明。

> **约定**
> - 文中所有“偏移”均为从文件起始字节开始的 **0 基偏移量**。
> - `int16`/`int32`/`uint8`/`double` 均采用 **小端序（little-endian）**。
> - 字符串字段如无特殊说明，编码为 **GBK**。
> - 技术原文《MapGIS 文件解析》中部分偏移以 1 基或区间形式描述（如字段描述符“从第 349 个字节开始”），本文统一换算为 0 基偏移（字段描述符起始于 348）。

---

## 1. 文件总体结构

三种文件均采用相同的顶层组织方式：

| 区域 | 说明 |
|------|------|
| 文件头 | 8 字节魔数 + 4 字节文件标识 + 4 字节索引区起始偏移 |
| 索引区 | 10 条记录，每条 10 字节，记录各子数据区的起始偏移与大小 |
| 子数据区 | 坐标/线/区/属性等实际数据 |

索引区起始偏移位于文件偏移 **12–15**，记为 `data_start`。从 `data_start` 开始连续存放 10 条索引记录：

```
head_1 : 偏移 data_start + 0
head_2 : 偏移 data_start + 10
...
head_10: 偏移 data_start + 90
```

每条索引记录前 8 字节为 `(start, volume)`，分别表示对应子数据区的起始偏移和大小（字节）。具体哪条 head 对应哪种数据，见后续点/线/区文件章节。

三种文件的文件头标识与文件标识值如下：

| 文件类型 | 魔数 | 文件标识 | 说明 |
|----------|------|----------|------|
| 点文件 `.wt` | `WMAP`D22` | 1 | 点要素 |
| 线文件 `.wl` | `WMAP`D21` | 0 | 线要素 |
| 区文件 `.wp` | `WMAP`D23` | 2 | 多边形要素 |

## 2. 坐标系解析

三种文件的坐标系参数存储位置和结构完全一致，主要位于文件头固定偏移处：

| 内容 | 偏移 | 字节数 | 类型 | 说明 |
|------|------|--------|------|------|
| 投影类型 | 109 | 1 | uint8 | 投影类型索引 |
| 椭球体类型 | 110 | 1 | uint8 | 椭球体索引 |
| 比例尺 | 143 | 8 | double | 比例尺分母 |

> 说明：投影类型索引与椭球体索引分别对应 MapGIS 安装目录下的投影索引表和 `ellip.dat` 中的椭球体索引。因此坐标系的完整解释依赖于外部软件环境。

### 2.1 常用椭球体

| 索引 | 大地基准 | PROJ4 定义 |
|------|----------|-----------|
| 1 | 北京 54 | `+ellps=krass +towgs84=15.8,-154.4,-82.3,0,0,0,0 +units=m +no_defs` |
| 2 | 西安 80 | `+a=6378140 +b=6356755.288157528` |
| 7 | WGS84 | `+datum=WGS84` |
| 16 | Krassovsky | `+ellps=krass` |

### 2.2 投影参数

高斯-克吕格（投影索引 5）、兰伯特（3）、阿尔伯斯（2）等投影的附加参数存储如下。这些偏移与《MapGIS 文件解析》中的描述一致，其中 175/183/191 三个偏移分别对应起始纬线、第一纬线圈、第二纬线圈：

| 内容 | 偏移 | 字节数 | 类型 | 说明 |
|------|------|--------|------|------|
| 中央经线 | 151 | 8 | double | 格式 `DDDMMSS.sss` |
| 起始纬线 | 175 | 8 | double | 标准平行圈 0 |
| 第一纬线圈 | 183 | 8 | double | 标准平行圈 1 |
| 第二纬线圈 | 191 | 8 | double | 标准平行圈 2 |
| 东偏移量 | 199 | 8 | double | False Easting |
| 北偏移量 | 207 | 8 | double | False Northing |

中央经线与标准纬线采用 `DDDMMSS.sss` 的 BCD 式双精度编码，需按度分秒拆分后换算为十进制度：

```
deg = s[:-4]
min = s[-4:-2]
sec = s[-2:]
decimal_degrees = deg + min/60 + sec/3600
```

对于经纬度（投影类型 0），坐标值为十进制度；对于投影坐标系，内部坐标通常以 **毫米** 为单位，读取后需除以比例尺分母再乘以 1000（即 `scale = scale / 1000`）转换为米。

## 3. 点文件（*.wt）

### 3.1 文件头与索引区

| 内容 | 偏移 | 字节数 | 类型 | 说明 |
|------|------|--------|------|------|
| 文件头标识 | 0–7 | 8 | string | `WMAP`D22` |
| 文件标识 | 8–11 | 4 | int32 | 固定为 1 |
| 索引区起始偏移 | 12–15 | 4 | int32 | `data_start` |

索引区（从 `data_start` 开始）中各 head 含义：

| head | 偏移（相对 data_start） | 含义 |
|------|------------------------|------|
| head_1 | 0 | 坐标数据区起始偏移与大小 |
| head_2 | 10 | （未使用） |
| head_3 | 20 | 属性数据区起始偏移与大小 |

### 3.2 坐标数据区

坐标数据区每 **93 字节** 存储一个点。**第一个 93 字节为空记录**，因此实际点数为：

```
point_count = coord_volume / 93 - 1
```

单个点记录结构：

| 内容 | 相对点记录偏移 | 字节数 | 类型 | 说明 |
|------|----------------|--------|------|------|
| 标注 | 0 | 1 | uint8 | 点标注 |
| 保留 | 1–6 | 6 | - | 未使用 |
| X 坐标 | 7–14 | 8 | double | X 坐标值 |
| Y 坐标 | 15–22 | 8 | double | Y 坐标值 |
| 保留 | 23–92 | 70 | - | 未使用 |

## 4. 线文件（*.wl）

### 4.1 文件头与索引区

| 内容 | 偏移 | 字节数 | 类型 | 说明 |
|------|------|--------|------|------|
| 文件头标识 | 0–7 | 8 | string | `WMAP`D21` |
| 文件标识 | 8–11 | 4 | int32 | 固定为 0 |
| 索引区起始偏移 | 12–15 | 4 | int32 | `data_start` |

索引区中各 head 含义：

| head | 偏移（相对 data_start） | 含义 |
|------|------------------------|------|
| head_1 | 0 | 线数据区起始偏移与大小 |
| head_2 | 10 | 坐标数据区起始偏移与大小 |
| head_3 | 20 | 属性数据区起始偏移与大小 |

### 4.2 线数据区

线数据区每 **57 字节** 存储一条线的索引信息，**第一个 57 字节为空记录**，实际线数为：

```
line_count = line_volume / 57 - 1
```

单条线索引结构：

| 内容 | 相对线索引偏移 | 字节数 | 类型 | 说明 |
|------|----------------|--------|------|------|
| 保留 | 0–9 | 10 | - | 未使用 |
| 锚点数目 | 10–13 | 4 | int32 | 该线包含的坐标点数量 |
| 锚点坐标存储偏移 | 14–17 | 4 | int32 | 相对于坐标数据区起始位置的偏移 |
| 保留 | 18–56 | 39 | - | 未使用 |

### 4.3 坐标数据区

坐标数据区按顺序存放所有线的锚点坐标，每对坐标占 **16 字节**（2 × double）。读取时根据线索引中的 `锚点数目` 和 `锚点坐标存储偏移` 定位并读取。

## 5. 区文件（*.wp）

### 5.1 文件头与索引区

| 内容 | 偏移 | 字节数 | 类型 | 说明 |
|------|------|--------|------|------|
| 文件头标识 | 0–7 | 8 | string | `WMAP`D23` |
| 文件标识 | 8–11 | 4 | int32 | 固定为 2 |
| 索引区起始偏移 | 12–15 | 4 | int32 | `data_start` |

索引区中各 head 含义：

| head | 偏移（相对 data_start） | 含义 |
|------|------------------------|------|
| head_1 | 0 | 弧段数据区起始偏移与大小 |
| head_2 | 10 | 坐标数据区起始偏移与大小 |
| head_3 | 20 | （未使用） |
| head_4 | 30 | 区拓扑数据区起始偏移与大小 |
| head_10 | 90 | 属性数据区起始偏移与大小 |

### 5.2 弧段数据区

区文件复用线的弧段存储结构：每 **57 字节** 一条弧段索引，**第一个 57 字节为空记录**。字段含义与 `.wl` 的线数据区完全一致（锚点数目 + 锚点坐标偏移）。

### 5.3 坐标数据区

与线文件相同，每 16 字节存放一对 double 坐标。区文件的所有弧段坐标集中存放在此区域。

### 5.4 区拓扑数据区

拓扑数据区每 **24 字节** 一条记录，**第一个 24 字节为空记录**。实际拓扑记录数为：

```
topo_count = poly_volume / 24 - 1
```

每条记录前 16 字节包含 4 个 int32，解析器主要使用其中第 3、4 个整数（即记录内偏移 8–15）：

| 内容 | 相对记录偏移 | 字节数 | 类型 | 说明 |
|------|--------------|--------|------|------|
| 左所属区号 | 8–11 | 4 | int32 | 弧段左侧的区 ID |
| 右所属区号 | 12–15 | 4 | int32 | 弧段右侧的区 ID |
| 保留 | 16–23 | 8 | - | 未使用 |

> 说明：区 ID 从 1 开始编号。解析时，选取左区号或右区号等于目标区 ID 的所有弧段，根据区 ID 位于左侧还是右侧决定是否需要反向，然后将弧段按端点连接成闭合环，再区分外环与内洞构造 `Polygon`/`MultiPolygon`。

### 5.5 区重建要点

1. **筛选弧段**：对目标区 ID `pid`，筛选 `left_id == pid` 或 `right_id == pid` 的弧段。
2. **定向**：若 `left_id == pid`，通常需要将该弧段坐标反向，以保证区边界始终位于弧段同一侧。
3. **成环**：采用贪心最近端点合并策略，将弧段连接为闭合环。端点坐标可能因浮点精度存在约 `1e-6` 级别的偏差。
4. **洞识别**：通过环之间的包含关系（point-in-polygon）判断外环与内洞，构造带洞多边形。
5. **有效性修复**：由于弧段合并可能产生自相交环，建议在重建后使用 `shapely.make_valid()` 等几何修复手段，使输出满足 OGC 简单要素规范。

## 6. 属性数据区结构（通用）

点、线、区文件的属性表结构一致。

### 6.1 属性表头

| 内容 | 相对属性区偏移 | 字节数 | 类型 | 说明 |
|------|----------------|--------|------|------|
| 保留 | 0–321 | 322 | - | 日期、工作目录等元数据 |
| 字段数 | 322–323 | 2 | int16 | `field_count` |
| 记录数 | 324–327 | 4 | int32 | `record_count` |
| 单条记录大小 | 328–329 | 2 | int16 | `record_length` |
| 保留 | 330–347 | 18 | - | 未使用 |
| 字段描述符 | 348 起 | 39 × field_count | - | 每个字段 39 字节 |

> 注：技术原文《MapGIS 文件解析》将字段描述符起始位置记为“第 349 个字节”，这是 1 基表述；本文采用 0 基偏移，因此起始于 **348**。

### 6.2 字段描述符

每个字段描述符固定 **39 字节**，结构如下：

| 内容 | 相对字段描述符偏移 | 字节数 | 类型 | 说明 |
|------|--------------------|--------|------|------|
| 字段名称 | 0–19 | 20 | string | GBK 编码，以 `\x00` 结尾 |
| 字段类型 | 20 | 1 | uint8 | 0–7 |
| 存储位置 | 21–24 | 4 | int32 | 字段在记录内的起始偏移 |
| 保留 / 显示宽度 | 25–38 | 14 | - | 含字段显示宽度、小数位等元数据 |

字段类型映射：

| 类型值 | 含义 | Python 类型 |
|--------|------|-------------|
| 0 | string | `str` |
| 1 | byte | `int` |
| 2 | short integer | `int` |
| 3 | integer | `int` |
| 4 | float | `float` |
| 5 | double | `float` |
| 6 | date | `datetime.date` |
| 7 | time | `datetime.time` |

### 6.3 记录数据

字段描述符之后紧跟 `record_count × record_length` 字节的数据。**第一条记录为空记录**，实际有效记录数为 `record_count - 1`。

每条记录按字段描述符中的 `存储位置` 切片读取。字段长度计算方式：

- 非末尾字段：`next_offset - current_offset`
- 末尾字段：`record_length - current_offset`

字符串字段读取到第一个 `\x00` 为止；日期字段为年（int16）+ 月 + 日；时间字段为时 + 分 + 秒（double，含微秒）。

## 7. 已知限制与注意事项

1. **坐标系不完整**：文件仅保存投影/椭球索引号，完整 CRS 需依赖外部 `ellip.dat` 及投影索引表。当前实现内置了北京 54、西安 80、WGS84 等常见椭球体，以及高斯-克吕格、阿尔伯斯、兰伯特等投影。
2. **区拓扑容错**：弧段端点可能存在微小偏差，且部分源数据本身存在拓扑缝隙或自相交。读取后建议调用几何有效性修复。
3. **编码**：字段名与字符串属性值使用 GBK 编码，若存在非法字节通常截断至首个非法字节处。
4. **比例尺**：投影坐标系下内部坐标常以毫米为单位，需按 `scale / 1000` 换算为米；经纬度坐标系下 `scale` 通常为 1。
